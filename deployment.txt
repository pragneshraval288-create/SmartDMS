## âš™ï¸ 4. Environment Configuration

1. **Create a `.env`** file in the project root.  
2. **Populate** it based on the template below:

```
# .env
SECRET_KEY=your_strong_secret_key_here
DATABASE_URL=mysql+pymysql://db_user:db_password@localhost/smartdms

FLASK_ENV=development
```
> **Note:** Never commit `.env` to version control.

---

## ðŸ—„ï¸ 5. MySQL Database Setup

1. **Login** to MySQL as a privileged user:  
   ```sql
   mysql -u root -p
   ```
2. **Create the database**:  
   ```sql
   CREATE DATABASE smartdms CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
3. **(Optional) Create a dedicated user**:  
   ```sql
   CREATE USER 'smartdms_user'@'localhost' IDENTIFIED BY 'strong_password';
   GRANT ALL PRIVILEGES ON smartdms.* TO 'smartdms_user'@'localhost';
   FLUSH PRIVILEGES;
   ```
4. Exit MySQL:  
   ```sql
   EXIT;
   ```

---

## ðŸ”„ 6. Database Migrations

SmartDMS uses Flask-Migrate (Alembic) to manage schema changes:

1. **Initialize migrations** (only the first time):  
   ```bash
   flask db init
   ```
2. **Generate a migration**:  
   ```bash
   flask db migrate -m "Initial schema"
   ```
3. **Apply migrations**:  
   ```bash
   flask db upgrade
   ```
